Git

Branches:

Developers can use it branching to deviate from the production version of code in order to correct an issue or add a feature. Developers construct branches. So they can work on a duplicate of the code without changing the original. We make branches to separate our code changes and test them before merging them into the main branch.

The main branch has nothing unique to offer. When we use the “git init” command to create a Git repository, it creates the first branch.

We can creat many branches in Git. We can give random names to it also. However, we need to define one branch as a main/master branch default in our repository.

Merge:

The git merge command allows you to combine the several development lines established by git branch into a single branch.

Git merge combines numerous commit sequences into a single unified history. Git merge is used to merge two branches in the most common usage situations.

For Instance, we have a new branch feature based on the main branch. This feature branch will now be merged into main. Calling “Merge” command merges the specified branch functionality into the current branch. Consider it as main. Git automatically determines the merge algorithm.

Commit:

we use “Commit “command to save our changes to the local repository. We should always add a message when we commit.

Git assigns a unique SHA-1 hash to each snapshot of files when we make a commit. Git produces a new pointer to the same commit that the main branch is currently on when we create a branch. Both branches have the identical code snapshot at this time.

By adding clear messages to each commit an pushing it to the Github, it is easy for everyone to see what has changed in the code and when. “--amend” option help us to rewrite the most commit with any changes that have been staged and/or a new commit message.

Tags:

Tags are references (refs) which point to a particular portion of a Git repository's history.

Tags offer assistance in distinguishing distinctive commits that are vital sufficient to be recognized or be taken note. For case, Tagging a commit with discharge adaptation 3.0 implies that commit was the ultimate commit some time recently the dispatch of the 3.0 adaptation of the computer program. In addition, it is helpful and simple. GitHub gives an entire diverse segment to see at the tags and analyze them there.

There are two types of tags. One is Annotated tag and another one is Light-weighted tag. Both of these tags are similar, but they are different in terms of the amount of Metadata stores.

Repository:

A Git repository is the .git/ folder internal a project. This repository tracks all modifications made to documents on our project, constructing a records over time.

Version Control System is used to create these versions for work and store them in a specific place(in our compute) termed as a repository.

We can delete and create a new repository for number of times regarding project.
